# 路线图

1. 稳定 CLI 使用体验：继续打磨安装、配置、错误提示、任务时间线与交互式流程。
2. 强化工具安全边界：补齐文件编辑、路径校验、通用 Shell 审批、进程中断清理和敏感字段脱敏。
3. 补强评测闭环：沉淀真实 coding 任务，关联评测结果、失败样本与任务执行记录，形成可复现实验指标。
4. 改进任务连续性：围绕现有 summary、todo 和 evidence replay 增加跨 session 的任务/证据检索。
5. 优化子代理委派：继续收紧 worker agent 的边界、结果摘要和失败处理；仅在独立子任务场景评估并行与文件冲突控制。
6. 根据真实任务需要，再评估声明式 Skills/MCP 扩展、规划/反思步骤或更重的上下文检索机制。
